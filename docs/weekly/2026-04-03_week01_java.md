# 2026-04-03 week01 java

## 1. 本周目标

本周目标是把 `media-task-platform-java` 从 0 起到一个可运行、可测试、可展示的最小 Spring Boot 工程骨架，而不是提前进入数据库、鉴权和平台化扩展。

## 2. 本次完成内容

本次已完成：

- 使用 Spring Initializr 生成 Maven + Spring Boot 项目骨架
- 完成 Java 21 环境补齐，并通过 Maven Wrapper 验证构建环境
- 统一主类为 `MediaTaskApplication`
- 增加 `HealthController`
- 暴露 `/health` 与 `/actuator/health`
- 增加最小 `MediaTask` 内存版 CRUD：
  - 创建任务
  - 查询任务列表
  - 按 ID 查询任务
  - 按 ID 删除任务
- 增加 `SmokeTest`
- 增加 `MediaTaskControllerTest`
- 验证 `./mvnw test` 通过
- 验证应用可启动并可通过 curl 访问接口

## 3. 当前工程判断

当前仓库已经具备“最小可运行后端骨架”的条件，但仍然只是第一阶段雏形。

当前优点：

- 工程入口清晰
- 健康检查可用
- Web 层和 Service 层已初步分离
- 有自动化测试，不是纯手点 demo
- 后续可自然演进到数据库和安全层

当前局限：

- 任务数据仅保存在内存中，重启即丢失
- 错误响应结构仍较简化
- 没有 repository 层
- 没有 migration
- 没有鉴权
- 没有 metrics / tracing
- 没有容器化与部署脚本
