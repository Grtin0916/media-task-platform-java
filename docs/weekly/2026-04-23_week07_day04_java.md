# 2026-04-23 Week07 Day04 Java

## 1. Goal

把 Java 仓从“Week07 eventing skeleton 已落盘”推进到“本地最小 event-flow smoke 已留证”的状态。

## 2. What was verified

本次本地 smoke 覆盖以下动作：

- 启动 PostgreSQL 与 Redis 容器
- 启动 Spring Boot 应用
- `POST /api/media-tasks` 创建任务并触发事件写入 stream
- `POST /api/media-tasks/eventing/smoke-consume` 触发 group 创建与最小消费
- 使用 `XRANGE`、`XINFO GROUPS`、`XPENDING` 校验 stream / group / ack 状态

## 3. Verified findings

本次确认：

- 应用、PostgreSQL、Redis 本地链路已正常启动
- `POST /api/media-tasks` 成功返回 `CREATED` 状态任务
- `media-task-events` stream 中可见 `MediaTaskCreated` 事件
- `media-task-group` consumer group 已被创建
- 在 group 已存在后再次创建任务，再执行 smoke consume，得到：
  - `CONSUMED eventName=MediaTaskCreated ... acked=1`
- `XPENDING` 为 `0`
- `XINFO GROUPS` 显示：
  - `consumers = 1`
  - `entries-read = 2`
  - `lag = 0`

## 4. Caveat

当前实现存在一个 Week07 边界：

- 如果消息先进入 stream，而 consumer group 尚未创建
- 第一次 `smoke-consume` 可能得到 `NO_MESSAGE`
- 在当前实现下，group 创建后的新消息可以被正常 consume + ack

因此，当前已验证的是：

- append / consume / ack 最小闭环成立

当前尚未验证的是：

- 建组前 backlog 的统一消费语义
- 更完整的任务状态更新闭环
- 幂等、重试、补偿与失败恢复
